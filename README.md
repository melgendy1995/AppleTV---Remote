# Apple TV Remote — Swift native app (macOS + iOS)

SwiftUI app that replaces the Chrome extension / PWA frontend with a native macOS and iOS UI. Talks to the same Python backend (FastAPI + pyatv) — pyatv's protocol implementations are too involved to port, so the backend stays.

## Layout

```
swift-app/
├── project.yml                    # XcodeGen config (multi-platform targets)
├── Entitlements-macOS.plist
├── Sources/
│   ├── AppleTVRemoteApp.swift     # @main, scene, command menu
│   ├── Models/Models.swift        # DeviceInfo, PlayingState, KeyboardEvent, ...
│   ├── Networking/BackendClient.swift  # REST + WebSocket client (ObservableObject)
│   └── Views/
│       ├── Theme.swift            # dark palette, button styles
│       ├── ContentView.swift      # root view + sheets (devices, pair, keyboard, settings)
│       ├── NowPlayingCard.swift
│       ├── RemoteControlsView.swift   # D-pad + menu + transport + system
│       ├── DevicePickerView.swift
│       ├── PairingView.swift      # session-based pair flow with PIN
│       ├── KeyboardView.swift     # auto-opens when TV text field is focused
│       └── SettingsView.swift     # backend URL with test-connection
└── AppleTVRemote.xcodeproj        # generated by XcodeGen
```

## Generate the Xcode project

```bash
brew install xcodegen        # one-time
cd swift-app
xcodegen generate            # whenever project.yml changes
open AppleTVRemote.xcodeproj
```

## Build / run

In Xcode: pick the scheme (`AppleTVRemote-macOS` or `AppleTVRemote-iOS`) and a destination, then Cmd+R.

Or from the CLI:

```bash
# macOS
xcodebuild -project AppleTVRemote.xcodeproj -scheme AppleTVRemote-macOS \
  -destination 'platform=macOS' -derivedDataPath build \
  CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO build
open "build/Build/Products/Debug/Apple TV Remote.app"

# iOS Simulator
xcodebuild -project AppleTVRemote.xcodeproj -scheme AppleTVRemote-iOS \
  -destination 'platform=iOS Simulator,name=iPhone 15' build
```

## Backend setup

The app talks to the FastAPI backend defined in `backend/`. Default URL is `http://localhost:8000`. Change it from the gear icon in the app.

On iOS, set it to the LAN IP of the Mac running the backend (e.g. `http://192.168.1.20:8000`).

## What works

- Device discovery, pairing (PIN flow inline), connect / disconnect
- D-pad, Menu (prominent), play/pause/prev/next, Home, Control Center
- Live now-playing card via WebSocket
- Auto-pop keyboard modal when a TV text field is focused; debounced text push, clear, submit
- Settings: editable backend URL with test-connection
- Cmd+, opens Settings on macOS

## Not yet

- Bundling Python backend inside the Mac .app (currently the user runs `uvicorn` separately).
- Code-signing / notarization config (the project builds unsigned for local dev).
- Haptics on iOS for D-pad presses (stubbed but ungated by Info.plist).

## Adding to the App Store later

You'd need to:
1. Set `DEVELOPMENT_TEAM` in `project.yml` to your Apple ID team ID.
2. Re-run `xcodegen generate`.
3. Archive in Xcode, distribute via App Store Connect.

Plus a non-trivial decision: how the App Store version reaches a backend. Most likely you'd let users enter a URL (current behavior), or bundle a small `pyatv`-using helper for macOS via Apple-signed Python — at which point it stops being App-Store-distributable.
